# 🎵 Music Stream Forecasting System
Система машинного обучения для прогнозирования музыкальных стримов, оценки доходности каталога и моделирования жизненного цикла треков и артистов.

## 📖 Описание проекта
Разработка и внедрение предиктивной модели для оценки количества будущих стримов с учётом динамики, жанровых особенностей и фаз жизненного цикла артистов/треков. Решение покрывает 4 ключевых бизнес-сценария:
1. ✅ **Существующие артисты** (каталог с историей 3–8 лет)
2. ✅ **Существующие артисты** (новые релизы)
3. 🔄 **Новые артисты** (ограниченная статистика < 1 года)
4. 🚧 **Cold Start** (новые артисты без истории, прогноз на основе поиска аналогов)

> **Бизнес-цель:** Этап 1 — точный прогноз стримов. Этап 2 — финансовое моделирование (авансы, роялти, выкуп прав) на основе рассчитанных стримов и stream rates.

## 🛠 Методология и Архитектура
### 📊 Подготовка данных
- Очистка, фильтрация выбросов и нормализация временной оси (`day 0` от релиза)
- Агрегация по фазам жизненного цикла (рост, плато, спад)
- Валидация полноты метаданных и стримовой статистики (Яндекс Музыка, ВК, Звук)

### 🤖 Feature Engineering & Modeling
| Блок | Описание |
|------|----------|
| **🎵 Аудио-признаки** | PCA-сжатие 12 аудио-характеристик (`energy`, `danceability`, `valence`, `tempo` и др.) в 3 главные компоненты |
| **📈 Временные признаки** | Лаги (`streams_lag_1`), скользящие средние (7d), волатильность, календарные фичи, `track_age_days` |
| **🌍 Тренды рынка** | 3 модели `Prophet` для прогнозирования общего тренда стримов, `danceability` и `energy` |
| **👤 Профили артистов** | Кластеризация по уровням (`Niche`, `Rising`, `Established`, `Superstar`) на основе пиковых и 95-перцентильных стримов |
| **🤖 Алгоритм** | **LightGBM** + автоматический подбор гиперпараметров через **Optuna** |
| **🔍 Explainability** | Анализ важности признаков (Feature Importance) для прозрачности прогнозов |

### 📈 Валидация и Backtesting
- Итеративный прогноз на 365 дней для тестовой выборки
- Метрики: `WMAPE`, `Median MAPE`, `RMSE`, `R²`, `Cumulative Error`
- Проверка на ретроспективных данных и совместная валидация с бизнес-логикой

## 📊 Результаты модели (Бэктест)
| Метрика | Значение | Интерпретация |
|:---|:---|:---|
| **WMAPE** | `27.02%` | Взвешенная средняя абсолютная ошибка |
| **Медианная MAPE** | `26.76%` | Ошибка по половине треков ниже этого порога |
| **R²** | `0.815` | Модель объясняет ~81.5% вариации стримов |
| **RMSE** | `225,981` | Среднеквадратичное отклонение в стримах |
| **Cumulative Error** | `-17.75%` | Небольшое систематическое занижение совокупного прогноза |
